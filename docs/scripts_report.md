
# VX Web Demo 스크립트 분석 보고서 (Scripts Report)

이 보고서는 프로젝트의 `package.json`에 정의된 주요 스크립트들의 용도, 사용 상황, 그리고 도출되는 결과물에 대해 상세히 설명합니다.

## 1. 프로젝트 생명주기 및 안정성 체크 (Lifecycle & Safety)

### `npm run prebuild`
- **사용 상황**: 프로젝트 빌드(`npm run build`)를 실행하기 직전에 자동으로 실행됩니다.
- **결과**: i18n 키 정합성, 360 뷰어 안정성, 도움말(Help) 경로 유효성을 전수 검사합니다. 하나라도 실패하면 빌드가 중단되어 불량 빌드 배포를 방지합니다.

### `npm run help:check` / `help:i18n-check`
- **사용 상황**: 도움말/Q&A 페이지와 관련된 파일, 라우터 설정 및 i18n 구조가 유효한지 확인하고 싶을 때 사용합니다.
- **결과**: 
  - `help:check`: `QA.md`, `HelpQaView.vue`, 각 언어별 `help.json` 파일의 존재 여부와 라우터 설정을 검증합니다.
  - `help:i18n-check`: `HelpQaView.vue`가 원본 마크다운을 직접 임포트하지 않고 `vue-i18n`의 `tm()` 함수를 통해 구조화된 데이터를 사용하는지 검사합니다.

### `npm run viewer360:check` / `viewer360:check-lifecycle`
- **사용 상황**: 360 뷰어(`Viewer360Live.vue`)의 코드를 수정했을 때, 메모리 누수나 라이프사이클 오류가 없는지 검사하기 위해 사용합니다.
- **결과**: 
  - `onUnmounted`가 비동기 `onMounted` 내부에 잘못 정의되었는지 확인합니다.
  - Three.js 리소스 해제(`dispose`) 및 정리 함수 존재 여부를 확인합니다.
  - 비동기 작업 시 `isActive` 가드 사용 여부를 체크합니다.

### `npm run routeguide:check`
- **사용 상황**: 경로 안내(Route Guide) 컴포넌트의 안정성을 검증할 때 사용합니다.
- **결과**: 컴포넌트 라이프사이클 관리와 리소스 정리가 제대로 구현되어 있는지 확인합니다.

## 2. 국제화 및 번역 관리 (i18n Management)

### `npm run i18n:check`
- **사용 상황**: 전체 언어팩의 일관성을 검토할 때 사용합니다.
- **결과**: 마스터 언어인 한국어(`ko`)를 기준으로 다른 언어 파일들에 빠진 키가 있는지, 한국어가 유출(leakage)되었는지, 혹은 `[TODO]` 표시가 남아있는지 전수 조사합니다.

### `npm run translate:locales`
- **사용 상황**: 한국어(`ko`) 로케일 파일에 새로운 문구를 추가한 후, 다른 언어(en, ja, zh-TW, es, ru)로 자동 번역 및 동기화할 때 사용합니다.
- **결과**: OpenAI GPT-4o API를 사용하여 맥락에 맞는 번역을 수행하고 각 언어별 `.json` 파일을 업데이트합니다. 번역 메타데이터는 `.translation-meta.json`에 기록됩니다.
  - `:todo`: `[TODO]` 표시가 있거나 번역이 비어있는 항목만 선별하여 번역합니다.
  - `:force`: 기존 번역 여부와 상관없이 전체를 다시 번역합니다.

### 영역별 검증 스크립트 (`i18n:validate-map`, `i18n:validate-events`, 등)
- **사용 상황**: 특정 기능 영역(지도, 이벤트, 알림 등)에 대한 로케일 데이터와 템플릿을 한꺼번에 검증할 때 사용합니다.
- **결과**: 해당 영역의 Vue 컴포넌트에서 하드코딩된 한국어를 스캔(`scan-*`)하고, 동시에 해당 영역의 `.json` 데이터 정합성(`check-*`)을 함께 확인합니다.

### `npm run i18n:validate-json`
- **사용 상황**: 언어팩 JSON 파일들에 문법 오류가 없는지 빠르게 확인할 때 사용합니다.
- **결과**: 모든 로케일 디렉토리의 JSON 파일들에 대해 `JSON.parse`를 수행하여 구문 오류를 찾아냅니다.

## 3. 데이터 검증 (Data Validation)

### `npm run events:validate`
- **사용 상황**: `events.json` 파일의 내부 데이터 구조가 유효한지 검사할 때 사용합니다.
- **결과**: 요일(day), 시간 형식(HH:mm), 카테고리 유효성, 상태값(status) 등이 사전에 정의된 규칙을 따르는지 검증합니다.

### `npm run events:verify`
- **사용 상황**: JSON 데이터가 원본 텍스트 문서(`docs/CO-pgm26_KO_*.txt`)와 일치하는지 최종 확인학 때 사용합니다.
- **결과**: 원본 텍스트를 파싱하여 JSON의 항목 수, 시간, 제목, 서브 항목들이 원본과 정확히 일치하는지 대조 분석합니다.

## 4. 분석 및 정리 (Analysis & Cleanup)

### `npm run analyze:deps`
- **사용 상황**: 사용되지 않는 라이브러리나 누락된 종속성을 찾을 때 사용합니다.
- **결과**: `depcheck`를 실행하여 `package.json`의 종속성 상태를 보고합니다.

### `npm run analyze:graph`
- **사용 상황**: 프로젝트의 모듈 간 의존성 구조를 시각적으로 파악하고 싶을 때 사용합니다.
- **결과**: `madge`를 통해 소스 코드의 의존성 그래프를 생성합니다.

### `npm run cleanup:check`
- **사용 상황**: 배포 전이나 코드 정리 후, 프로젝트가 깨끗한 상태인지 최종 확인할 때 사용합니다.
- **결과**: 전체 빌드를 수행한 후 i18n 무결성 검사를 실행하여 프로젝트의 건전성을 보장합니다.
