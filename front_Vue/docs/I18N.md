# 런타임 i18n 시스템 가이드 (Runtime i18n System Guide)

## 개요 (Overview)
VX Web V2는 국제 컨벤션을 위해 설계되었습니다. 정확하고 상황에 맞는 번역을 동적으로 제공하는 것은 핵심 요구사항입니다. 이 애플리케이션은 `vue-i18n`을 기반으로 한 **런타임 안전 i18n 아키텍처**를 활용합니다.

## Master-Mirror 정책 (Master-Mirror Policy)
로컬라이징 불일치를 방지하기 위해, 시스템은 한국어(`ko`) 로케일을 기준으로 엄격한 스키마를 강제합니다.

*   **단일 진실 공급원 (Source of Truth)**: `src/i18n/locales/ko/`
*   **미러 (Mirrors)**: `en`, `es`, `ja`, `zh-TW`, `ru`
*   **규칙**: 어떤 미러 파일도 `ko`에 없는 키를 가질 수 없으며, `ko`에 있는 키를 누락할 수도 없습니다.

## 하네스 검증 스크립트 (Harness Validation Scripts)
수동 확인에 의존하지 않고, 빌드 전 i18n 무결성을 보장하는 자동화된 Node.js 스크립트를 포함하고 있습니다.

*   `scripts/check-i18n-keys.cjs`: 모든 언어의 JSON 파일을 `ko`와 재귀적으로 비교합니다. 키가 누락되었거나 한국어 텍스트가 다른 로케일로 유입(Leakage)된 경우 프로세스를 중단(Exit Code 1)합니다.
*   `scripts/sync_locales_keys.cjs`: 미러 로케일에 누락된 키를 자동으로 생성하고, 나중에 번역할 수 있도록 소스인 `ko` 텍스트를 플레이스홀더로 채워주는 유틸리티입니다.

## 개발자 베스트 프랙티스 (Best Practices for Developers)
1.  **하드코딩 금지**: Vue 템플릿에 원시 문자열(예: `<div>안녕하세요</div>`)을 직접 쓰지 마세요.
2.  **사용법**: 템플릿에서는 `$t('namespace.key')`를, `<script setup>`에서는 `const { t } = useI18n()`을 사용하세요.
3.  **중첩 컨텍스트**: Mock 데이터에서 동적 스팟 이름을 해석하는 것과 같은 복잡한 객체의 경우, `src/utils/i18nResolver.js`에 있는 `resolveI18nText` 유틸리티를 활용하세요.
4.  **언어 추가**: `src/i18n/locales/` 아래에 새 폴더를 만들고 `ko`의 구조를 복사한 뒤, 번역을 업데이트하고 `src/i18n/index.js`에 로케일을 등록하면 됩니다.

## 안전한 대체 기능 (Safe Fallbacks)
런타임에 키를 찾을 수 없는 경우, UI가 깨지지 않도록 `ko` 로케일로 자동 대체(Fallback)되도록 구성되어 있으며, 동시에 개발자에게 콘솔 경고를 로그로 남깁니다.
